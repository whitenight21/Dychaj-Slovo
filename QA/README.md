# QA Testing Portfolio – Dýchaj Slovo

**Live Site:** [https://whitenight21.github.io/Dychaj-Slovo/](https://whitenight21.github.io/Dychaj-Slovo/)

A complete manual + automation testing playground built on my personal spiritual/family website. All testing follows ISTQB CTFL principles in an Agile/Scrum environment.

### Project Overview
- **Type**: Static multi-page spiritual website (HTML/CSS/JS + future Flask backend)
- **Domain**: Family devotionals, children’s activities, daily reflections, PDF products
- **Purpose**: Demonstrate end-to-end QA skills for Junior QA / Automation Tester roles
- **Current Status**: Manual testing + Jira + Git complete | Playwright automation & API testing in progress

### Repository Structure  
QA/  
    ├── Screenshots/  ← Jira execution & bug reports  
    ├── Test-Cases/  ← ISTQB-style manual test cases  
    ├── Bugs/  ← Detailed defect reports  
    ├── Automation/   ← Playwright Python scripts  
    ├── API-Testing/  ← Postman collections  
    └── README.md   ← This file  
    
### Tools & Technologies Used
**Core QA Tools**
- Jira (bug tracking, test cases, sprints)
- Playwright + Python + Pytest (web automation)
- Git & GitHub (version control + CI/CD pipelines)
- Postman (API testing – in progress)

**Additional Skills Demonstrated**
- Agile/Scrum (sprints in Jira)
- STLC & SDLC (full testing lifecycle)
- Accessibility & responsiveness testing
- Form validation, session management, multi-language support

### Testing Scope Covered
- **Manual Testing**: Functional, regression, smoke, exploratory, UAT, boundary, negative
- **Non-functional**: Performance, accessibility (contrast, screen reader), responsiveness (mobile-first)
- **Automation**: Playwright scripts for critical flows (login → cart → checkout)
- **API Testing**: Mock endpoints for newsletter and PDF delivery
- **Documentation**: All test cases and bugs tracked in Jira with screenshots
